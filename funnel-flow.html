Tabelas do Banco

Tabela Finalidade
funnels Funil raiz — produto, status, proprietário
funnel_pages Páginas do funil (upsell, downsell, checkout, thankyou…)
checkout_offers Ofertas vinculadas a uma página (funnel_id + page_id)
user_product_access ✅ Onde o acesso ao produto é gravado
offer_analytics Eventos por oferta: view / accepted / decline
funnel_analytics Eventos por página: view / conversion / purchase

Liberação de Acesso — user_product_access

Tipo de produto Coluna preenchida Conflito UPSERT
app_product product_id user_id, product_id
application application_id user_id, application_id
member_area member_area_id user_id, member_area_id

Caminhos de Pagamento

  • 1-click (one_click_purchase = true): usa stripe_payment_method_id salvo → /api/process-upsell
  • Novo checkout: redireciona para /checkout/:url com offer_id e purchase_id

Sequência resumida

# Etapa Handler / Tabela
1 Produtor configura funil e páginas no painel funnels · funnel_pages · checkout_offers
2 Cliente compra o produto principal process-payment → user_product_access (salva cartão Stripe)
3 Página pós-compra carrega o script com purchase_id funnel-page-widget
4 Modal de upsell é exibido (evento view registrado) offer_analytics
5a Cliente aceita — 1-click → Stripe cobra off-session process-upsell → Stripe PaymentIntent
5b Cliente aceita — novo checkout → redireciona process-payment (fluxo normal)
6 Acesso ao produto do upsell liberado user_product_access (UPSERT)
7 Evento accepted registrado offer_analytics
